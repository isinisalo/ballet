# Ballet Agent Instructions

## UI Work

- Read `DESIGN.md` before making frontend or UI changes.
- Treat dark theme as the primary visual target and keep `Dark / Light / System` visible.
- Use the Ballet Matte Workbench style: compact forms, matte panels, subdued borders, filled controls, and restrained contrast.
- Do not expose technical debug UI in primary user flows unless the user explicitly asks for it.
- Do not add raw source panels, dependency lists, dry-run widgets, or disclosure sections to ordinary forms as a way to handle complexity.
- For visual changes, run rendered browser QA on desktop and mobile and check for overlap, horizontal overflow, console errors, and framework overlays.
